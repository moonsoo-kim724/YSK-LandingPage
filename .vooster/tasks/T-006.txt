# 예약 시스템(검사예약 포함) 및 실시간 캘린더 UI

**Task ID:** T-006
**Status:** DONE
**Importance:** MUST
**Complexity:** 7/10
**Urgency:** 9/10
**Dependencies:** T-002, T-005

## Description

### 설명
사용자가 직접 검사 및 상담 예약을 할 수 있는 실시간 캘린더 UI를 구현합니다. 사용자는 가능한 날짜와 시간을 선택하여 예약을 완료할 수 있으며, 예약 정보는 실시간으로 시스템에 반영됩니다.

### 구현 상세
1.  **캘린더 UI 컴포넌트 개발**: `shadcn/ui`의 `Calendar` 컴포넌트(내부적으로 `react-day-picker` 사용)를 기반으로 예약 날짜 선택 UI를 구현합니다.
2.  **실시간 가능 시간 조회 API**: 사용자가 특정 날짜를 선택하면, 해당 날짜의 예약 현황을 `reservations` 테이블에서 조회하여 예약 가능한 시간 슬롯 목록을 반환하는 Supabase Edge Function을 작성합니다.
3.  **예약 폼 및 제출 로직**: `react-hook-form`과 `zod`를 사용하여 예약자 정보(이름, 연락처 등)를 입력받는 폼을 만듭니다. 폼 제출 시, 선택된 날짜/시간과 사용자 정보를 `reservations` 테이블에 `INSERT`하는 함수를 호출합니다.
4.  **실시간 반영**: 예약이 성공적으로 완료되면, `zustand` 같은 상태 관리 라이브러리를 통해 UI를 즉시 업데이트하여 사용자에게 예약 완료 상태를 알리고, 다른 사용자들이 해당 시간 슬롯을 선택할 수 없도록 합니다. (Supabase Realtime을 구독하여 다른 사용자에게도 즉시 반영되도록 고도화 가능)

### 테스트 전략
1.  **시간 슬롯 확인**: 특정 날짜를 선택했을 때, 이미 예약된 시간이 비활성화되어 선택할 수 없는지 확인합니다.
2.  **예약 생성 테스트**: 테스트 사용자로 로그인하여 예약을 생성하고, `reservations` 테이블에 해당 데이터가 올바르게 기록되는지 확인합니다.
3.  **중복 예약 방지 테스트**: 동일한 시간 슬롯에 대해 두 명의 사용자가 거의 동시에 예약을 시도하는 경우, 한 명의 예약만 성공적으로 처리되는지 확인합니다.
4.  **UI/UX 테스트**: 캘린더와 시간 선택 과정이 직관적이고 사용하기 편리한지, 모바일 환경에서도 문제없이 동작하는지 확인합니다.
