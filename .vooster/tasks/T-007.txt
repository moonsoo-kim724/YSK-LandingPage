# 지도/위치 안내 및 네비게이션 연동

**Task ID:** T-007
**Status:** DONE
**Importance:** MUST
**Complexity:** 4/10
**Urgency:** 8/10
**Dependencies:** T-001

## Description

### 설명
사용자가 병원 위치를 쉽게 찾을 수 있도록 지도와 길찾기 기능을 제공합니다. 지도 임베딩을 통해 시각적인 위치 정보를 제공하고, '길찾기' 버튼 클릭 시 사용자의 휴대폰으로 위치 정보 링크를 전송하여 네비게이션 앱과 바로 연동되도록 합니다.

### 구현 상세
1.  **지도 API 연동**: 카카오맵 또는 네이버 지도 API의 JavaScript SDK를 사용하여 랜딩페이지 내 특정 섹션에 병원 위치가 표시된 지도를 임베드합니다.
2.  **'길찾기' 기능 구현**: '길찾기' 버튼을 만들고, 클릭 시 모달창을 통해 사용자에게 카카오톡 또는 문자로 길찾기 링크를 받을 수 있는 옵션을 제공합니다.
3.  **위치 링크 전송**: 사용자가 옵션을 선택하고 전화번호를 입력하면, Supabase Edge Function을 호출합니다. 이 함수는 Twilio(SMS) 또는 카카오톡 비즈메시지 API를 사용하여 카카오맵/네이버지도 길찾기 URL(딥링크 포함)을 담은 메시지를 전송합니다.
4.  **네비게이션 앱 연동**: 전송된 URL은 사용자가 클릭 시 스마트폰에 설치된 지도 앱(카카오맵, 네이버지도 등)을 바로 실행하여 길찾기를 시작하도록 구성합니다. (예: `kakaomap://route?ep=...`)

### 테스트 전략
1.  **지도 표시 확인**: 페이지 로드 시 지정된 위치에 지도가 정확하게 표시되는지 확인합니다.
2.  **링크 전송 테스트**: '길찾기' 기능을 통해 테스트 전화번호로 메시지를 발송하고, 5초 이내에 정상적으로 수신되는지 확인합니다.
3.  **딥링크 테스트**: 수신된 메시지 내의 링크를 클릭했을 때, 스마트폰에서 해당 네비게이션 앱이 자동으로 실행되고 목적지가 올바르게 설정되는지 확인합니다.
4.  **UI/UX 테스트**: 길찾기 기능의 사용 과정이 사용자에게 명확하고 간편한지 검토합니다.
