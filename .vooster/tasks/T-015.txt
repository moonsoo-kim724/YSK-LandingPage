# 코드래빗(coderabbit.ai) 연동 및 코드 리팩토링 자동화

**Task ID:** T-015
**Status:** DONE
**Importance:** MUST
**Complexity:** 7/10
**Urgency:** 10/10
**Dependencies:** T-012

## Description

### 설명
코드래빗(coderabbit.ai) API 키를 활용하여 GitHub Actions와 연동하고, 코드 리뷰, 리팩토링 제안, 테스트 코드 생성을 자동화하여 코드 품질 관리 프로세스를 구축합니다. 이를 통해 개발 생산성을 높이고 코드베이스의 일관성을 유지합니다.

### 구현 상세
1.  **GitHub Secrets 설정**: 제공된 코드래빗 API 키(`cr-bcc24cbc338ab0e70a0dd1e4f86534e179ececd7f22c25ede18af6c327`)를 프로젝트 GitHub 저장소의 `Settings > Secrets and variables > Actions`에 `CODERABBIT_API_KEY`라는 이름으로 안전하게 저장합니다.
2.  **GitHub Actions 워크플로우 구성**: `.github/workflows/` 디렉토리에 `coderabbit_review.yml` 파일을 생성합니다. Pull Request가 생성되거나 업데이트될 때마다 실행되도록 `on: [pull_request]` 트리거를 설정합니다.
    ```yaml
    name: CodeRabbit AI Review
    on: [pull_request]
    jobs:
      review:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v3
          - uses: coderabbitai/ai-pr-reviewer@latest
            with:
              github_token: ${{ secrets.GITHUB_TOKEN }}
              coderabbit_api_key: ${{ secrets.CODERABBIT_API_KEY }}
              openai_api_key: ${{ secrets.OPENAI_API_KEY }} # 필요한 경우
    ```
3.  **코드래빗 설정 파일 추가**: 프로젝트 루트에 `.coderabbit.yml` 파일을 생성하여 리뷰 규칙을 커스터마이징합니다. 예를 들어, 특정 파일 경로는 리뷰에서 제외하거나, 리뷰 요약, 코드 개선 제안 등 특정 기능의 활성화 여부를 설정할 수 있습니다.
    ```yaml
    # .coderabbit.yml
    language: "ko"
    early_access: true
    reviews:
      request_changes_indicator: "🚨"
      high_level_summary: true
      poem: false
    ```
4.  **자동화 프로세스 통합**: 설정된 워크플로우를 통해 새로운 PR이 생성되면 코드래빗이 자동으로 코드 변경 사항을 분석하고, 잠재적 버그, 스타일 위반, 개선점을 찾아 리뷰 코멘트로 제안합니다. 또한, PR 설명에 대한 요약문을 자동으로 생성하여 리뷰어의 이해를 돕도록 합니다.

### 테스트 전략
1.  **기능 검증용 PR 생성**: 간단한 코드 수정(예: 변수명 변경, 로직 수정)을 포함하는 테스트 브랜치를 생성하고 main 브랜치로 Pull Request를 생성합니다.
2.  **자동 리뷰 확인**: PR 페이지에서 GitHub Actions가 정상적으로 실행되고, 잠시 후 코드래빗 봇(`@coderabbitai`)이 리뷰 코멘트와 PR 요약을 자동으로 추가하는지 확인합니다.
3.  **리팩토링 제안 검증**: 코드래빗이 제안한 리팩토링 내용이나 수정 사항이 타당한지 검토하고, 실제로 적용했을 때 문제가 없는지 로컬 환경에서 테스트합니다.
4.  **CI/CD 연동 테스트**: 코드래빗의 자동화된 프로세스가 기존의 CI/CD 파이프라인(T-012)과 충돌 없이 원활하게 통합되어 작동하는지 확인합니다.
