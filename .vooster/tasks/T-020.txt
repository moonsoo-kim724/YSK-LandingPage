# 관리자용 데이터 시각화 대시보드 UI 리디자인

**Task ID:** T-020
**Status:** BACKLOG
**Importance:** SHOULD
**Complexity:** 6/10
**Urgency:** 6/10
**Dependencies:** T-014, T-016

## Description

### 설명
관리자 페이지(T-014)에 있는 데이터 시각화 대시보드의 UI를 리디자인하여 데이터의 가독성과 신뢰성을 높입니다. 최신 UI 트렌드를 반영한 카드, 그래프, 컬러 시스템을 적용하여 관리자가 핵심 성과 지표(KPI)를 빠르고 직관적으로 파악할 수 있도록 개선합니다.

### 구현 상세
1.  **차트 라이브러리 스타일링**: `Recharts` 또는 `Chart.js`와 같은 기존 차트 라이브러리의 스타일을 새 디자인 가이드에 맞춰 커스터마이징합니다. 그래프의 색상 팔레트, 폰트, 툴팁 디자인 등을 `T-016`에서 정의한 디자인 토큰을 사용하여 수정합니다.
2.  **카드 기반 레이아웃 도입**: 주요 지표(총 리드 수, 예약 전환율, 챗봇 사용 횟수 등)를 한눈에 파악할 수 있도록 각각의 지표를 카드 형태의 컴포넌트로 디자인합니다. 각 카드에는 수치, 변화율, 간단한 차트를 함께 표시합니다.
3.  **데이터 테이블 가독성 개선**: 대시보드 내 데이터 테이블의 UI를 개선합니다. 행(Row)마다 미세한 배경색 차이를 주어 가독성을 높이고, 정렬 가능한 헤더, 페이지네이션 컴포넌트 등을 새 디자인에 맞게 통일합니다.
4.  **전체 레이아웃 재구성**: 대시보드 전체를 CSS Grid 또는 Flexbox를 사용하여 논리적인 정보 그룹(예: 사용자 유입, 예약 현황, 챗봇 통계)으로 재구성합니다. 이를 통해 관리자가 원하는 정보를 더 쉽게 찾을 수 있도록 정보 구조를 명확화합니다.

### 테스트 전략
1.  **데이터 정확성 검증**: 리디자인된 대시보드에 표시되는 모든 수치와 그래프가 Supabase DB의 원본 데이터(T-014)와 일치하는지 교차 확인합니다.
2.  **차트 렌더링 테스트**: 다양한 데이터(예: 데이터가 없는 경우, 데이터가 매우 많은 경우)에 대해 모든 차트가 깨지거나 오류 없이 정상적으로 렌더링되는지 확인합니다.
3.  **인터랙티브 기능 테스트**: 날짜 필터, 데이터 정렬, 검색 등 대시보드 내의 모든 인터랙티브 기능이 정상적으로 작동하고, 그 결과가 차트와 테이블에 올바르게 반영되는지 테스트합니다.
4.  **시각적 일관성 검증**: 대시보드에 사용된 모든 컴포넌트(카드, 차트, 테이블, 버튼 등)가 `T-016`에서 정의한 디자인 시스템과 일관성을 유지하는지 시각적으로 검토합니다.
